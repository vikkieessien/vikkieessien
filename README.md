**Senior Data Analyst & Analytics Engineer** based in Berlin.

I build the data infrastructure that makes revenue and growth metrics trustworthy — from raw, messy source data through clean staging layers to production-ready models that finance, product, and growth teams can actually rely on.

- **Revenue analytics** — NRR, GRR, cohort retention, MRR waterfall, CAC payback
- **Growth & product analytics** — funnel conversion, CLV, feature adoption, user segmentation
- **Analytics engineering** — dbt model design (staging → intermediate → marts), data quality testing, and building pipelines that don't break silently

## Featured work

### [cohort-nrr-project](https://github.com/vikkieessien/cohort-nrr-project)
End-to-end dbt project modeling cohort-based **Net Revenue Retention (NRR)** and **Gross Revenue Retention (GRR)** from raw CRM and billing data — including a full customer-month spine so churn shows up as explicit $0 rows, a revenue waterfall with a built-in reconciliation check, and NRR/GRR sliced by segment and region. Runs fully locally on DuckDB.

> Demonstrates: multi-layer dbt architecture · messy data handling · metric definition · 86 automated tests
